# Git Cleanup & Pre-Commit Hooks

This directory contains scripts to help maintain a clean git repository by automatically managing temporary files, test results, and build artifacts.

---

## 📋 Scripts Overview

| Script | Purpose | When to Use |
|--------|---------|-------------|
| `cleanup-before-commit.ps1` | Interactive cleanup of temporary files | Before first commit, or when repository gets messy |
| `install-git-hooks.ps1` | Install automatic pre-commit checks | After cloning/initializing repository |
| `pre-commit.ps1` | Automatic pre-commit validation | Runs automatically via git hook (don't call directly) |

---

## 🚀 Quick Start

### First Time Setup

```powershell
# 1. Initialize git repository (if not already done)
git init

# 2. Install pre-commit hooks
.\install-git-hooks.ps1

# 3. Clean up existing temporary files
.\cleanup-before-commit.ps1

# 4. Review changes
git status

# 5. Add files and commit
git add .
git commit -m "Initial commit"
```

The pre-commit hook will automatically check for issues before each commit!

## 🧹 Cleanup Script Usage

### Interactive Mode (Recommended)
```powershell
.\cleanup-before-commit.ps1
```
Prompts you for each category of files found. Options:
- **[D]** Delete all files in category
- **[I]** Add directory to .gitignore and keep files
- **[K/S]** Keep files as-is (skip)

### Dry Run Mode (Preview Only)
```powershell
.\cleanup-before-commit.ps1 -DryRun
```
Shows what would be done without making any changes.

### Automatic Mode (No Prompts)
```powershell
.\cleanup-before-commit.ps1 -Auto
```
Automatically handles all temporary files according to predefined rules.

## 📂 What Gets Cleaned Up?

### 1. **JMeter Logs** → DELETE
- `jmeter.log`
- `jmeter_*.log`
- **Reason:** Generated during test execution, not needed in git

### 2. **JTL Test Results** → ADD TO .gitignore
- `demo-tool/logs/log_*.jtl`
- `demo-tool/logs/test_*.jtl`
- **Exception:** `phase1_success_baseline.jtl` is kept (reference baseline)
- **Reason:** Test results change frequently, bloat repository

### 3. **Application Logs** → ADD TO .gitignore
- `demo-tool/logs/*.log`
- **Reason:** Runtime logs, not source code

### 4. **Maven Build Artifacts** → DELETE + .gitignore
- `demo-tool/target/`
- **Reason:** Generated by Maven, can be rebuilt

### 5. **JMX Test Artifacts** → DELETE
- `demo-tool/src/main/resources/jmx/*.log`
- `demo-tool/src/main/resources/jmx/*.jtl`
- **Reason:** Test artifacts shouldn't be in source tree

### 6. **Temp/Backup Files** → DELETE
- `*.tmp`, `*.temp`, `*.bak`, `*~`
- **Reason:** Editor backups and temporary files

### ✅ Baseline Test Data
- `logs/phase1_success_baseline.jtl` - Reference test result for validation

## 🔒 Pre-Commit Hook

Once installed with `.\install-git-hooks.ps1`, the pre-commit hook automatically:

### ✅ Checks Before Each Commit
1. **Log files** - Warns if staging `*.log` or `*.jtl` files (except baselines)
2. **Temporary files** - Blocks staging `*.tmp`, `*.temp`, `*.bak`
3. **Build artifacts** - Warns if staging `target/` directory files

### 🚫 If Issues Found
```
❌ Pre-commit check failed!

The following issues were found:
  ⚠ Log/result file staged: demo-tool/logs/log_20260506_130315_sip-001.jtl
  ⚠ Build artifact staged: demo-tool/target/classes/Demo.class

Recommendations:
  1. Run: .\cleanup-before-commit.ps1
  2. Unstage unwanted files: git reset HEAD <file>
  3. Or force commit: git commit --no-verify
```

### ⚠️ Bypass Hook (Use Sparingly)
```powershell
git commit --no-verify -m "Your message"
```

## 🎯 Common Workflows

### Before First Commit
```powershell
# Clean up temporary files
.\cleanup-before-commit.ps1

# Install hooks
.\install-git-hooks.ps1

# Review what will be committed
git status

# Add and commit
git add .
git commit -m "Initial commit"
```

### Regular Development
```powershell
# Make changes to code
# ... edit files ...

# Stage changes
git add src/main/java/com/tts/demo/MyClass.java

# Commit (hook runs automatically)
git commit -m "Add new feature"

# If hook fails, clean up:
.\cleanup-before-commit.ps1
git add .
git commit -m "Add new feature"
```

### After Running Tests
```powershell
# Tests generate logs and results in demo-tool/logs/

# Option 1: Clean up automatically
.\cleanup-before-commit.ps1 -Auto

# Option 2: Keep logs but ignore them
# (already in .gitignore, so just commit your code changes)
git add src/
git commit -m "Update test code"
```

### Repository Maintenance
```powershell
# See what's untracked
git status

# Clean up manually
.\cleanup-before-commit.ps1

# Or use git's built-in clean (CAREFUL - deletes untracked files!)
git clean -fdX  # Remove ignored files only
git clean -fdn  # Dry run - show what would be deleted
```

## 🔧 Troubleshooting

### "Pre-commit hook not running"
```powershell
# Check if hook is installed
Test-Path .git\hooks\pre-commit

# Reinstall if missing
.\install-git-hooks.ps1 -Force
```

### "Git not initialized"
```powershell
# Initialize git
git init

# Then install hooks
.\install-git-hooks.ps1
```

### "Cleanup script says files exist but I don't see them"
Files might be in subdirectories or hidden. Use:
```powershell
# See all files matching pattern
Get-ChildItem -Recurse -File -Include *.log, *.jtl
```

### "I want to commit a log file for documentation"
Use one of these approaches:

**Option 1:** Force add
```powershell
git add -f demo-tool/logs/important_baseline.jtl
git commit --no-verify -m "Add reference baseline"
```

**Option 2:** Update .gitignore to exclude specific file
```
# In .gitignore, add exception:
!demo-tool/logs/important_baseline.jtl
```

## 📚 Best Practices

### ✅ DO
- Run `cleanup-before-commit.ps1` before your first commit
- Let the pre-commit hook run automatically
- Keep baseline/reference test files (e.g., `phase1_success_baseline.jtl`)
- Review `git status` before committing
- Commit test scripts (`test-*.ps1`) and documentation (`*.md`)

### ❌ DON'T
- Commit `target/` directory
- Commit timestamped log files (`log_20260506_*.jtl`)
- Commit temporary files (`*.tmp`, `*.bak`)
- Bypass pre-commit hook unless necessary (`--no-verify`)
- Delete test/validation scripts (`test-*.ps1`)
